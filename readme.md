# 🧪 Mini-Labs: Experimental Playground

## 💡 Why I Built This
This repository is my "coding laboratory." It houses smaller-scale projects born out of curiosity—experiments that aren't quite big enough for their own massive repository but are too valuable to lose. Whether it's for boosting my daily productivity or just scratching a technical itch, this is where I test, fail, and iterate.

## 🛠️ What It Does
This is a collection of various tools and scripts. 
- **Productivity:** Small utilities to optimize my daily workflow.
- **Experimental Sandbox:** A place to "give a shot" at new ideas and technologies.
- **Growth Log:** Documentation of my journey as a developer, keeping track of what I've built and learned along the way.

## 🏗️ Architecture
The structure is kept modular so that I can easily scale as I add more labs:

```text
Mini-labs/
├── README.md
├── requirements.txt
├── .gitignore
└── Labs/
    ├── 01_Monthly_Calendar/
    │   ├── db_manager.py
    │   └── monthly_calendar.py
    └── 02_Coming_Soon/
```
